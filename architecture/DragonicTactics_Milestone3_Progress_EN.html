üé® Design Pattern Implementation

Factory Pattern for Character Creation Kinam, Taegyeong

Centralized character creation with data-driven configuration

  • ‚Ä¢ CharacterFactory: Returns std::unique_ptr<Character>
  • ‚Ä¢ Data-driven: Character stats loaded from characters.json
  • ‚Ä¢ Type safety: Uses CharacterTypes enum (Dragon, Fighter, Cleric, Wizard, Rogue)
  • ‚Ä¢ Consistency: All characters created through factory, ensuring proper component attachment

Smart Pointer Ownership Model Taegyeong

Established clear ownership semantics using RAII and smart pointers

  • ‚Ä¢ Ownership: GameObjectManager owns characters via std::unique_ptr
  • ‚Ä¢ References: Other systems (GridSystem, TurnManager) use raw pointers for non-owning references
  • ‚Ä¢ Transfer: std::move() for ownership transfer from factory to manager
  • ‚Ä¢ Safety: No memory leaks, clear responsibility, compile-time enforcement

AI Strategy Pattern Implementation Sangyoon, Taegyeong, Seungju

Implemented Strategy pattern for AI decision-making with Fighter as first complete strategy

  • ‚Ä¢ Interface: IAIStrategy defines MakeDecision() contract
  • ‚Ä¢ Fighter flowchart: Polished Mermaid diagram ‚Üí fully implemented in code
  • ‚Ä¢ Complex logic: Treasure carrying, HP management, pathfinding, attack targeting
  • ‚Ä¢ Extensibility: ClericStrategy, WizardStrategy, RogueStrategy planned for GAM250
