Architecture Overview

Dragonic Tactics is built on a custom C++20 game engine with a modular, component-based architecture. The engine separates core systems (Engine layer), rendering abstraction (CS200 layer), low-level graphics (OpenGL layer), and game-specific logic (DragonicTactics layer).

This architecture follows key principles: modular independence, event-driven communication, data-driven design, and component-based composition.

Data Flow & Communication Patterns

Event-Driven Communication

Loosely-coupled systems communicate via EventBus (Game/DragonicTactics/StateComponents/EventBus.h/cpp)

â€¢ Publisher: CombatSystem publishes CharacterDamagedEvent

â€¢ Subscriber: UI Manager subscribes to show damage text

â€¢ File: Game/DragonicTactics/Types/Events.h defines all event types

Direct Communication

Tightly-coupled systems pass references directly

â€¢ Example: PlayerInputHandler receives GridSystem* and CombatSystem*

â€¢ Reason: Immediate gameplay operations require fast access

â€¢ File: Game/DragonicTactics/States/GamePlay.cpp:Update()

Data-Driven Architecture

Game content loaded from JSON files

â€¢ Characters: Assets/Data/characters.json

â€¢ Maps: Assets/Data/maps.json

â€¢ Parser: External/json.hpp (nlohmann/json)
